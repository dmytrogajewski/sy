# BUG-20260513-2336: Super+A claude agent silently no-ops (claude-agent-acp not executable)

## Summary
Super+A launcher accepts a prompt but nothing happens when the chosen agent is `claude`; `goose` works.

## Reproduction
- Method: manual
- Command: `claude-agent-acp --version` (direct invocation, same as the daemon does)
- Evidence:
  ```
  $ /home/dmitriy/.local/bin/claude-agent-acp --version
  bash: /home/dmitriy/.local/bin/claude-agent-acp: Permission denied
  $ echo $?
  126
  ```
  while `/usr/bin/env node /home/dmitriy/.local/lib/node_modules/@agentclientprotocol/claude-agent-acp/dist/index.js --version` exits 0.

  `ls -la` on the symlink target:
  ```
  -rw-r--r--. … dist/index.js     # mode 644 — no +x
  ```
  `goose` is `-rwxr-xr-x`, hence unaffected.

## Expected
Choosing `claude` from the Super+A picker, typing a prompt, and pressing Enter starts a session and dispatches the prompt (a notify-send confirmation fires, waybar updates).

## Actual
`sy-agentd` `start_session` calls `AcpChild::spawn(&spec, cwd)` → `tokio::process::Command::spawn("claude-agent-acp")` → execve returns EACCES because `dist/index.js` lacks +x. The daemon returns `ClientReply::Error { … }` to `sy agt run`, which prints to stderr — but Super+A spawns the launcher via `niri spawn`, so there is no TTY and the error is lost. Hence "nothing happens".

## Root Cause
`@agentclientprotocol/claude-agent-acp@0.32.0` ships `dist/index.js` without the executable bit. The package.json `bin.claude-agent-acp = dist/index.js` only creates a symlink in `~/.local/bin/`; it does not chmod the target. The shebang is valid (`#!/usr/bin/env node`) but the kernel refuses to exec a file without `+x`. `sy apply`'s `ensure_bridges` runs `npm install -g @agentclientprotocol/claude-agent-acp` and never compensates for this — so the system drifts off `sy` the moment npm leaves the bit clear.

Per the CLAUDE.md "no snowflakes" rule the fix lives in `sy`, not in a one-off `chmod` on the host.

## Fix
1. Add `ensure_bin_executable(prefix, pkg)` to `src/main.rs` that reads the installed package's `package.json`, walks each `bin` entry, and sets `mode | 0o111` on the target if the bit is missing.
2. Call it unconditionally at the end of `ensure_bridges` (also when the installed version already matches latest), so drift heals on every `sy apply`.
3. As an immediate side effect we restore the bit on the currently installed `dist/index.js` so the user is unblocked the moment they run `sy apply`.

Out of scope for this bug (filed separately): make the launcher surface daemon spawn errors via `notify-send` when stdin is not a TTY, so a future regression is loud instead of silent.

## Traceability
- Failing repro before fix: `/home/dmitriy/.local/bin/claude-agent-acp --version` → exit 126, "Отказано в доступе".
- After fix (patched `sy apply` ran): same command → exit 0. File mode now `-rwxr-xr-x`.
- Fixed in: `src/main.rs` — added `ensure_bin_executable(prefix, pkg)` (~30 lines) and call it both *before* the version check (so a transient npm install failure still leaves a working launcher) and *after* a successful reinstall.
